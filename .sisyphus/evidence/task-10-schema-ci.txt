# Task 10: Schema Repo CI Validation and PR Enforcement

## Summary
Created GitHub Actions workflows and PR templates to enforce schema change validation and artifact freshness in the camperplaner-schema repository.

## Files Created

### 1. /.github/workflows/validate-migrations.yml
Validates migration files on push and PR:
- **Naming Convention**: Enforces `YYYYMMDDhhmmss_description.sql` pattern
- **SQL Syntax**: Basic syntax validation (valid statements, parentheses balance)
- **Timestamp Ordering**: Ensures chronological order, no duplicates
- **Conflict Markers**: Checks for unresolved git merge conflicts

### 2. /.github/workflows/check-artifacts.yml
Validates generated artifacts on migration changes:
- **Schema Manifest**: Verifies `generated/schema-manifest.json` exists and is valid JSON
- **Manifest Sync**: Checks that manifest's latest migration matches filesystem
- **TypeScript Types**: Verifies `generated/database.types.ts` exists and exports types
- **Documentation Linkage**: Ensures CONTRACT.md exists and references latest migration
- **Checksums**: Validates artifact checksums in manifest
- **PR Comments**: Automatically comments on PRs if artifacts need update

### 3. /.github/PULL_REQUEST_TEMPLATE.md
PR template with comprehensive checklist:
- Schema change type selection
- Pre-submit validation checklist
- Artifact generation requirements
- Consumer impact documentation
- Post-deploy verification steps

## Workflow Triggers

Both workflows trigger on:
- **Push** to any branch (when migrations change)
- **Pull Request** (when migrations or artifacts change)

Paths monitored:
- `supabase/migrations/*.sql`
- `generated/**`

## Validation Flow

1. Developer creates migration file with correct naming
2. Developer runs `node scripts/generate-types.js` to update artifacts
3. Developer commits migration + updated artifacts
4. Developer opens PR with template checklist
5. CI runs validation checks
6. If checks pass, PR can be merged
7. If checks fail, PR is blocked with automated comments
