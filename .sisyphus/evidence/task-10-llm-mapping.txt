================================================================================
SECTION 3: TRUST SCORE DERIVATION
================================================================================

Formula:
  trust_score = confidence * (1 - COALESCE(hallucination_risk, 0))

Rationale:
- confidence (0-1): How confident the LLM is in its output
- hallucination_risk (0-1): Risk of fabricated information
- Combined: High confidence + low hallucination risk = high trust
- Range: 0.0 (no trust) to 1.0 (full trust)

Edge cases:
- If confidence IS NULL → trust_score = NULL
- If hallucination_risk IS NULL → treat as 0 (no penalty)
- If both NULL → trust_score = NULL

SQL expression:
  CASE WHEN ple.confidence IS NOT NULL
       THEN ple.confidence * (1 - COALESCE(ple.hallucination_risk, 0))
       ELSE NULL
  END AS trust_score

================================================================================
SECTION 4: SOURCE URLS AGGREGATION
================================================================================

All source URLs from place_llm_sources for the enrichment are aggregated into
a JSONB text array stored in source_urls.

Aggregation logic:
  SELECT jsonb_agg(source_url ORDER BY relevance_score DESC NULLS LAST, id)
  FROM place_llm_sources
  WHERE llm_enrichment_id = <enrichment_id>

Ordering rationale:
- relevance_score DESC: Most relevant URLs first
- NULLS LAST: URLs without relevance score come after scored ones
- id ASC: Stable ordering for ties

Edge cases:
- No sources → source_urls = NULL (not empty array)
- Single source → single-element array
- Duplicate URLs → preserved as-is (deduplication is application concern)

================================================================================
SECTION 5: FACTS PIVOT STRATEGY
================================================================================

place_llm_facts stores facts as rows (field_name, value_text, value_type).
place_llm_properties stores facts as columns.

Pivot approach:
1. For each enrichment, collect all facts into a JSONB object:
   SELECT jsonb_object_agg(field_name, value_text)
   FROM place_llm_facts
   WHERE llm_enrichment_id = <enrichment_id>

2. In the INSERT, use COALESCE to extract each field:
   COALESCE(facts->>'name', NULL) AS name
   COALESCE(facts->>'has_electricity', NULL)::boolean AS has_electricity

3. Boolean parsing for value_type='boolean':
   CASE WHEN facts->>'has_electricity' IN ('true','1','yes') THEN true
        WHEN facts->>'has_electricity' IN ('false','0','no') THEN false
        ELSE NULL END

Alternative (simpler for batch):
  Use subquery per column:
  (SELECT value_text FROM place_llm_facts
   WHERE llm_enrichment_id = ple.id AND field_name = 'has_electricity'
   LIMIT 1) AS raw_has_electricity

Recommended: JSONB aggregation approach for single-pass efficiency.

CREATE OR REPLACE FUNCTION parse_bool(text)
RETURNS boolean AS $$
BEGIN
    RETURN CASE
        WHEN $1 IN ('true', '1', 'yes', 'TRUE', 'Yes') THEN true
        WHEN $1 IN ('false', '0', 'no', 'FALSE', 'No') THEN false
        ELSE NULL
    END;
END;
$$ LANGUAGE plpgsql IMMUTABLE;
