SECTION 1: MINIMAL PLACES TABLE DEFINITION
================================================================================

The `places` table MUST be reduced to ONLY these 6 columns:

Column      | Type        | Constraints              | Description
------------|-------------|--------------------------|---------------------------
id          | bigint      | PK, NOT NULL, GENERATED  | Unique place identifier
lat         | numeric     | NULL                     | Latitude (WGS84)
lon         | numeric     | NULL                     | Longitude (WGS84)
is_active   | boolean     | NOT NULL, DEFAULT true   | Active flag
created_at  | timestamptz | NOT NULL, DEFAULT NOW()  | Row creation time
updated_at  | timestamptz | NOT NULL, DEFAULT NOW()  | Last update time

================================================================================
SECTION 4: WHY THESE 6 COLUMNS
================================================================================

id:
  - Primary key, required for FK references from property tables
  - Unchanged from current schema

lat, lon:
  - Minimal spatial reference for map display
  - Replaces geom (PostGIS geometry) which is being removed
  - Sufficient for point-based queries without PostGIS dependency
  - Source-specific coordinates remain in property tables as source_lat/source_lon

is_active:
  - Soft-delete flag for place lifecycle management
  - Used by import system to mark stale/removed places
  - NOT business data - it's infrastructure

created_at, updated_at:
  - Standard audit timestamps
  - Required for data freshness tracking
  - NOT business data - it's infrastructure

================================================================================
SECTION 5: WHAT DOES NOT BELONG IN PLACES
================================================================================

BUSINESS DATA:
  - Names, descriptions → property tables
  - Addresses, contact info → property tables
  - Amenity flags → property tables
  - Place types → property tables

SOURCE METADATA:
  - source_primary → each source has its own property row
  - data_confidence → computed at read time from source resolution
  - last_seen_at → tracked per-source in property tables
  - last_enriched_at → tracked per-source in property tables

GEOMETRY:
  - geom → removed; lat/lon sufficient for minimal table
  - Full geometry remains in osm_source.geom if needed
