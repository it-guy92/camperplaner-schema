================================================================================

Target: place_google_properties (wide table)

--------------------------------------------------------------------------------
1.2 TYPE MAPPINGS (place_google_types → place_google_properties)
--------------------------------------------------------------------------------

| Source Column | Target Column      | Transform                                              |
|---------------|--------------------|--------------------------------------------------------|
| google_type   | source_place_type  | WHERE is_primary = true → single text value            |
| google_type   | source_categories  | ALL types → array_agg(google_type) as text[]           |

SQL Pattern:
  source_place_type = (SELECT google_type FROM place_google_types
                       WHERE google_source_id = src.id AND is_primary = true
                       LIMIT 1)
  source_categories = (SELECT array_agg(google_type ORDER BY is_primary DESC, google_type)
                       FROM place_google_types WHERE google_source_id = src.id)

--------------------------------------------------------------------------------
1.3 AMENITY MAPPINGS (place_google_amenities → place_google_properties)
--------------------------------------------------------------------------------

Each amenity_key maps to a boolean column via conditional aggregation.

| amenity_key        | Target Column            | Value Source    |
|--------------------|--------------------------|-----------------|
| electricity        | has_electricity          | value_boolean   |
| water              | has_fresh_water          | value_boolean   |
| wifi               | has_wifi                 | value_boolean   |
| shower             | has_shower               | value_boolean   |
| restrooms          | has_restrooms            | value_boolean   |
| laundry            | has_laundry              | value_boolean   |
| dump_station       | has_dump_station         | value_boolean   |
| pet_friendly       | pets_allowed             | value_boolean   |
| accessibility      | wheelchair_accessible    | value_boolean   |
| pool               | has_pool                 | value_boolean   |
| playground         | has_playground           | value_boolean   |
| restaurant         | has_restaurant           | value_boolean   |
| bar                | has_cafe                 | value_boolean   |
| shop               | has_shop                 | value_boolean   |
| beach              | has_beach                | value_boolean   |
| washing_machine    | has_laundry              | value_boolean   |
| dishwasher         | has_dishwashing_area     | value_boolean   |

SQL Pattern (conditional aggregation):
  MAX(CASE WHEN a.amenity_key = 'electricity' THEN a.value_boolean END) AS has_electricity
  MAX(CASE WHEN a.amenity_key = 'water' THEN a.value_boolean END) AS has_fresh_water
  ... (one CASE per amenity_key)

Note: 'washing_machine' and 'laundry' both map to has_laundry.
  Resolution: COALESCE(MAX(laundry), MAX(washing_machine)) — laundry takes precedence.

Note: 'bar' maps to has_cafe (closest semantic match in target schema).
  Rationale: Google 'bar' type indicates food/drink service; has_cafe is the target column.

================================================================================
SECTION 2: BATCHED BACKFILL STRATEGY
================================================================================

--------------------------------------------------------------------------------
2.1 BATCHING APPROACH
--------------------------------------------------------------------------------

Method: Process in batches of place_id ranges to avoid long-running transactions.

  Batch size: 1000 place_ids per batch
  Cursor: place_id ranges (MIN to MAX, chunked)
  Idempotency: ON CONFLICT (place_id) WHERE is_current = true DO UPDATE

Why batching:
- Prevents lock escalation on large tables
- Allows progress monitoring via RAISE NOTICE
- Enables resume on failure (process unprocessed place_ids)
- Keeps transaction log size manageable

--------------------------------------------------------------------------------
2.2 BACKFILL SQL TEMPLATE
--------------------------------------------------------------------------------

DO $$
DECLARE
  batch_size INTEGER := 1000;
  min_id BIGINT;
  max_id BIGINT;
  current_min BIGINT;
  current_max BIGINT;
  rows_affected INTEGER;
  total_rows INTEGER := 0;
BEGIN
  -- Get the range of place_ids that have Google sources
  SELECT MIN(place_id), MAX(place_id)
  INTO min_id, max_id
  FROM place_google_sources
  WHERE is_current = true;

  IF min_id IS NULL THEN
    RAISE NOTICE 'No Google sources found. Nothing to backfill.';
    RETURN;
  END IF;

  RAISE NOTICE 'Backfilling Google properties for place_ids % to %', min_id, max_id;

  current_min := min_id;

  WHILE current_min <= max_id LOOP
    current_max := current_min + batch_size - 1;

    INSERT INTO place_google_properties (
      place_id,
      is_current,
      source_updated_at,
      name,
      address,
      source_lat,
      source_lon,
      website,
      phone,
      -- Boolean amenity columns
      has_electricity,
      has_fresh_water,
      has_wifi,
      has_shower,
      has_restrooms,
      has_laundry,
      has_dump_station,
      pets_allowed,
      wheelchair_accessible,
      has_pool,
      has_playground,
      has_restaurant,
      has_cafe,
      has_shop,
      has_beach,
      has_dishwashing_area,
      -- Type columns
      source_place_type,
      source_categories,
      -- Google-specific columns
      google_source_id,
      google_place_id,
      rating,
      review_count,
      business_status,
      expires_at
    )
    SELECT
      src.place_id,
      true AS is_current,
      src.fetched_at AS source_updated_at,
      src.name,
      src.formatted_address AS address,
      src.lat AS source_lat,
      src.lon AS source_lon,
      src.website,
      src.phone,
      -- Amenity booleans via conditional aggregation
      MAX(CASE WHEN a.amenity_key = 'electricity' THEN a.value_boolean END) AS has_electricity,
      MAX(CASE WHEN a.amenity_key = 'water' THEN a.value_boolean END) AS has_fresh_water,
      MAX(CASE WHEN a.amenity_key = 'wifi' THEN a.value_boolean END) AS has_wifi,
      MAX(CASE WHEN a.amenity_key = 'shower' THEN a.value_boolean END) AS has_shower,
      MAX(CASE WHEN a.amenity_key = 'restrooms' THEN a.value_boolean END) AS has_restrooms,
      COALESCE(
        MAX(CASE WHEN a.amenity_key = 'laundry' THEN a.value_boolean END),
        MAX(CASE WHEN a.amenity_key = 'washing_machine' THEN a.value_boolean END)
      ) AS has_laundry,
      MAX(CASE WHEN a.amenity_key = 'dump_station' THEN a.value_boolean END) AS has_dump_station,
      MAX(CASE WHEN a.amenity_key = 'pet_friendly' THEN a.value_boolean END) AS pets_allowed,
      MAX(CASE WHEN a.amenity_key = 'accessibility' THEN a.value_boolean END) AS wheelchair_accessible,
      MAX(CASE WHEN a.amenity_key = 'pool' THEN a.value_boolean END) AS has_pool,
      MAX(CASE WHEN a.amenity_key = 'playground' THEN a.value_boolean END) AS has_playground,
      MAX(CASE WHEN a.amenity_key = 'restaurant' THEN a.value_boolean END) AS has_restaurant,
      MAX(CASE WHEN a.amenity_key = 'bar' THEN a.value_boolean END) AS has_cafe,
      MAX(CASE WHEN a.amenity_key = 'shop' THEN a.value_boolean END) AS has_shop,
      MAX(CASE WHEN a.amenity_key = 'beach' THEN a.value_boolean END) AS has_beach,
      MAX(CASE WHEN a.amenity_key = 'dishwasher' THEN a.value_boolean END) AS has_dishwashing_area,
      -- Types
      (SELECT t.google_type FROM place_google_types t
       WHERE t.google_source_id = src.id AND t.is_primary = true
       LIMIT 1) AS source_place_type,
      (SELECT array_agg(t.google_type ORDER BY t.is_primary DESC, t.google_type)
       FROM place_google_types t
       WHERE t.google_source_id = src.id) AS source_categories,
      -- Google-specific
      src.id AS google_source_id,
      src.google_place_id,
      src.rating,
      src.review_count,
      src.business_status,
      src.expires_at
    FROM place_google_sources src
    LEFT JOIN place_google_amenities a ON a.google_source_id = src.id
    WHERE src.is_current = true
      AND src.place_id BETWEEN current_min AND current_max
    GROUP BY src.id, src.place_id, src.name, src.formatted_address,
             src.lat, src.lon, src.website, src.phone,
             src.fetched_at, src.google_place_id, src.rating,
             src.review_count, src.business_status, src.expires_at
    ON CONFLICT (place_id) WHERE is_current = true
    DO UPDATE SET
      source_updated_at = EXCLUDED.source_updated_at,
      name = EXCLUDED.name,
      address = EXCLUDED.address,
      source_lat = EXCLUDED.source_lat,
      source_lon = EXCLUDED.source_lon,
      website = EXCLUDED.website,
      phone = EXCLUDED.phone,
      has_electricity = EXCLUDED.has_electricity,
      has_fresh_water = EXCLUDED.has_fresh_water,
      has_wifi = EXCLUDED.has_wifi,
      has_shower = EXCLUDED.has_shower,
      has_restrooms = EXCLUDED.has_restrooms,
      has_laundry = EXCLUDED.has_laundry,
      has_dump_station = EXCLUDED.has_dump_station,
      pets_allowed = EXCLUDED.pets_allowed,
      wheelchair_accessible = EXCLUDED.wheelchair_accessible,
      has_pool = EXCLUDED.has_pool,
      has_playground = EXCLUDED.has_playground,
      has_restaurant = EXCLUDED.has_restaurant,
      has_cafe = EXCLUDED.has_cafe,
      has_shop = EXCLUDED.has_shop,
      has_beach = EXCLUDED.has_beach,
      has_dishwashing_area = EXCLUDED.has_dishwashing_area,
      source_place_type = EXCLUDED.source_place_type,
      source_categories = EXCLUDED.source_categories,
      google_source_id = EXCLUDED.google_source_id,
      google_place_id = EXCLUDED.google_place_id,
      rating = EXCLUDED.rating,
      review_count = EXCLUDED.review_count,
      business_status = EXCLUDED.business_status,
      expires_at = EXCLUDED.expires_at,
      updated_at = now();

    GET DIAGNOSTICS rows_affected = ROW_COUNT;
    total_rows := total_rows + rows_affected;

    RAISE NOTICE 'Batch [% to %]: % rows affected (total: %)',
      current_min, current_max, rows_affected, total_rows;

    current_min := current_max + 1;
    COMMIT;  -- Release locks between batches (requires procedure, not DO block)
  END LOOP;

  RAISE NOTICE 'Backfill complete. Total rows affected: %', total_rows;
END $$;

2.3 PROCEDURE VERSION (for COMMIT between batches)
--------------------------------------------------------------------------------

CREATE OR REPLACE PROCEDURE backfill_google_properties(batch_size INTEGER DEFAULT 1000)
LANGUAGE plpgsql

  min_id BIGINT;
  max_id BIGINT;
  current_min BIGINT;
  current_max BIGINT;
  rows_affected INTEGER;
  total_rows INTEGER := 0;

  SELECT MIN(place_id), MAX(place_id)
  INTO min_id, max_id
  FROM place_google_sources
  WHERE is_current = true;

    RAISE NOTICE 'No Google sources found. Nothing to backfill.';

  RAISE NOTICE 'Backfilling Google properties for place_ids % to %', min_id, max_id;
  current_min := min_id;

    current_max := current_min + batch_size - 1;

      place_id, is_current, source_updated_at, name, address,

================================================================================
SECTION 3: TRANSFORM RULES SUMMARY
================================================================================

3.1 Identity Fields:
  - name: Direct from place_google_sources.name
  - address: Direct from place_google_sources.formatted_address
  - source_lat/lon: Direct from place_google_sources.lat/lon

3.2 Contact Fields:
  - website: Direct from place_google_sources.website
  - phone: Direct from place_google_sources.phone
  - email: Always NULL (Google doesn't provide)
  - opening_hours: NULL (not in normalized source; in raw_payload if needed)
  - fee_info: NULL (not in normalized source)

3.3 Amenity Booleans:
  - Each amenity_key → specific boolean column via MAX(CASE ... END)
  - NULL = no claim (Google didn't report this amenity)
  - COALESCE for overlapping keys (laundry/washing_machine → has_laundry)

3.4 Type Fields:
  - source_place_type: Primary type (is_primary=true) from place_google_types
  - source_categories: All types aggregated as text[] array

3.5 Google-Specific Fields:
  - google_source_id: FK to place_google_sources.id
  - google_place_id: Google's place identifier
  - rating: Google rating (0-5 scale)
  - review_count: Number of Google reviews
  - business_status: OPERATIONAL, CLOSED_TEMPORARILY, etc.
  - expires_at: Cache expiration timestamp

3.6 Metadata:
  - is_current: true (all backfilled rows are current)
  - source_updated_at: From place_google_sources.fetched_at
  - created_at/updated_at: Auto-generated by defaults/triggers

  amenity_key,
  COUNT(*) AS amenity_count,
  COUNT(*) FILTER (WHERE value_boolean = true) AS true_count
FROM place_google_amenities a
JOIN place_google_sources src ON src.id = a.google_source_id
WHERE src.is_current = true
GROUP BY amenity_key
ORDER BY amenity_key;
