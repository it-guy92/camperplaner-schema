### 3. Schema Design Patterns (Lines 947-989)

Add new pattern:

```markdown
### Property Table Pattern
Phase 2 introduces aligned property tables that replace source-family parent tables.

**Key Differences from Source Families:**
- Source families: Raw import data (osm_source, place_google_sources, place_llm_enrichments)
- Property tables: Aligned business data with 55 shared columns

**Read Resolution:**
```sql
SELECT COALESCE(
  user.column_name,
  osm.column_name,
  google.column_name,
  llm.column_name
) AS resolved_value
FROM places p
LEFT JOIN place_user_properties user ON user.place_id = p.id AND user.is_current = true
LEFT JOIN place_osm_properties osm ON osm.place_id = p.id AND osm.is_current = true
LEFT JOIN place_google_properties google ON google.place_id = p.id AND google.is_current = true
LEFT JOIN place_llm_properties llm ON llm.place_id = p.id AND llm.is_current = true
WHERE p.id = $1;
```

**Current-Row Semantics:**
- OSM/Google/LLM: One current row per place_id
- User: One current row per (place_id, user_id)
- Enforced via partial unique indexes: `WHERE is_current = true`
```
